# Risk Register

| ID | Severity | Area | Finding | Evidence | Impact | Recommendation | Acceptance |
| -- | -------- | ---- | ------- | -------- | ------ | -------------- | ---------- |
| R-001 | P0 | NBR readiness/security | Session-accessible NBR `/check` can trust client `image_present/docker_available` and set ready. | `internal/server/api/runtime_handlers.go` `HandleCheckNodeBackendRuntime`, `upsertNodeBackendRuntime(checkOnly=true)`. | Wrong deployment and security bypass of Agent-verified evidence. | Remove route, make it Agent-only, or force it to call server-side `/check-request` probe. | API test proves session callers cannot set ready from request body; only Agent/server probe evidence can. |
| R-002 | P1 | API contract/E2E | E2E scripts still use old deployment `backend_runtime_id` and `parameters_json`. | `scripts/e2e-matrix-verifier.sh`, `scripts/e2e-dryrun-parameter-matrix-enhanced.sh`, `scripts/e2e-model-runtime-wizard-nvidia-api.sh`. | API-first validation is not trustworthy. | Update scripts to current contract or archive them. | All active E2E scripts use `node_backend_runtime_id`, `/check-request`, and `parameter_values_json`; CI runs them or marks hardware skips. |
| R-003 | P1 | RunPlan boundary | `/deployments/preflight` is not the final RunPlan resolver path and rejects `ready_with_warnings` while create/start accept it. | `preflight_handlers.go` checks `status == "ready"` and does not call `preflightDeployment`; `deployment_lifecycle_handlers.go` uses `isNBRDeployable` and `runplan.Resolve`. | UI/API may greenlight configs that dry-run/start reject, or block warning-only NBRs that start would allow. | Make preflight call the same resolver/deployability helper or rename it as candidate check and add full pre-start preview API. | Contract tests show preflight/dry-run/start agree on deployable statuses and blocking errors. |
| R-004 | P1 | Frontend UX/API | Deployment edit runtime selector is misleading and ignored. | `ModelDeploymentsPage.vue` edit `backend_runtime_id`; `doEdit()` omits it. | User believes runtime changed when it did not. | Remove selector or implement explicit NBR change flow with snapshot warning. | Browser/API test confirms edit form only shows editable fields or successfully changes NBR with recheck. |
| R-005 | P1 | Snapshot integrity | Migrations and legacy branch mutate/rebuild frozen snapshots. | `db.go` snapshot updates; `runtime_handlers.go` legacy update branch. | Snapshot rules are not absolute; future bugs can hide in upgrade branches. | Remove no-compat paths or document one-time migrations with tests. | Fresh DB and current DB tests prove snapshots are only changed by explicit user action or named migration. |
| R-006 | P1 | API documentation | OpenAPI is stale and omits current routes. | `docs/api/openapi.yaml`. | External clients and agents target wrong APIs. | Regenerate/update OpenAPI. | OpenAPI includes deployments/NBR/runplans/model-roots and validates sample payloads. |
| R-007 | P1 | Agent security | One global Agent bearer token protects all agent APIs. | `AgentAuthMiddleware`, `cmd/agent/main.go` endpoints. | Token leak grants broad node/API access. | Add per-agent/node credentials or signed token binding. | Agent token cannot be reused by a different node ID; negative test covers mismatch. |
| R-008 | P1 | Docker security | Dangerous Docker options are configurable without policy gate. | `RuntimeParameterEditor`, `docker_real.go`. | Users can run privileged containers/mount host devices/paths. | Add policy and permission model for privileged options. | Non-admin tenant cannot enable privileged/devices/host networking unless policy allows. |
| R-009 | P2 | Tenant/schema cleanliness | `gpu_devices` default tenant remains `'default'` in schema creation paths. | `db.go`, `resource_handlers.go`. | Nonstandard initialization can create invalid tenant IDs. | Replace defaults with UUID default or require explicit tenant. | Fresh DB schema contains no `tenant_id DEFAULT 'default'`. |
| R-010 | P2 | Tests | Coverage weak in auth/authz/db/rbac/main/metrics. | `go test ./... -cover`. | Security and migrations can regress. | Add negative contract and migration tests. | Coverage plus targeted tests for critical route families. |
| R-011 | P2 | Performance | Frontend makes per-node NBR requests. | `ModelDeploymentsPage.vue` `loadAllNBRs()`. | More nodes cause page fan-out and latency. | Add aggregate NBR list endpoint. | One API call can fetch tenant-scoped NBRs with node/runtime metadata. |
| R-012 | P2 | Product scope | Multi-replica/distributed fields exist but start path is single-instance. | `HandleStartDeployment` inserts one instance and `mode='single'`. | Users may expect replicas to work. | Hide/disable replicas or implement scheduler. | API rejects replicas >1 until supported, or E2E verifies replicas. |
| R-013 | P2 | Observability | Server-managed Prometheus/Grafana supervision not implemented in Go. | Docs and scripts. | Ops story depends on external scripts/binaries. | Keep as documented limitation or implement supervisor. | `/observability/status` and scripts agree on supported mode. |
| R-014 | P2 | OpenAI compatibility | No API key/gateway/usage billing; only instance test path exists. | Router lacks `/v1/*` gateway/API key; docs mark future. | Cannot claim platform serving API maturity. | Keep out of current maturity claims. | Gateway/API key usage has explicit design and tests before advertised. |
| R-015 | P3 | Build performance | Frontend main chunk exceeds 500 kB warning. | `npm run build`. | Slower first load as UI grows. | Add manual chunks/code splitting. | Build warning resolved or documented threshold accepted. |
